#os.walk
```
os.walk(top[, topdown=True[, onerror=None[, fllowlinks=False]]])
	Generate the filename in a directory tree by walking the tree either top-down or bottom-up. For each directory in the tree rooted at directory top ( include top itself ), it yield a 3-tuple (dirpath, dirnames, filenames)

	dirpath is a string, the path to the directory. dirnames is a list of the names of the subdirectories in dirpath (excluding '.' and '..'). filenames is a list of names of the non-directory files in dirpath. Note that the names in the lists contain no path components. To get a full path(which begins with top) to a file or directory in dirpath, do os.path.join(dirpath, name).

	If optional argument topdown is True or not specified, the triple for a direcotry is generated before the triples for any of its subdirectory( directories are generated top-down ). If topdown is False the triple for a directory is generated after the triple for all of tis subdirecotries( directories are generated bottom-up ).

	when topdown is True, the caller can modify the dirnames list in-place(perhaps using del or slice assignment), and walk() will only recurse into to subdirectories whose names remain in dirnames; this can be used to prune the search, impose a specific order of visiting, or even to inform walk() about directories the caller creates or renames before it resumes walk() again. Modifiing direnames when topdown is False in ineffective, because in bottom-up mode the directories in direnames are generated before dirpath itself is generated.

	By default errors from the listdir() call are ignored. If optional argument onerror is specified. it should be a function; it will be called with one argument, an OSError instance. It can report the error to continue with the walk, or raise the exception to abort the walk. Note that the filename is available as the filename attribute of the exception object.

	By default, walk() will not walk down into symbolic links that resolve to directories. Set fllowlinks to True visit directories pointed to by symlinks, on systems that support them.
```

遍历给定的目录，把返回一个有３个元素的元组，(路径名，目录名，文件名)
- 路径名：类似于linxu下的bsedir
- 目录列表：把当前目录下的所有目录存放在一个列表中
- 文件列表：把当前目录下所有的文件存入在一个列表，如果没有则返回一个空列表
- 需要注意的是，os.walk这是一个生成器，和文件对象一样，在遍历完成后就没有了，如果要使用需要重新生成
- os.walk不处理'.'和'..'，但是可以显示出隐藏文件
