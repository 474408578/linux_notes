1. 有时候执行task任务后，服务器发生变化后需要执行一些操作，比如我们修改了配置文件，需要重启一下服务
    需要注意的问题：tasks中的notify后面的字符和handlers中的字符一样是为了起到关联作用，也就是说只有tasks中的任务执行成功才会执行handler中的伤
    必要条件：
        a、task中的notify的值和你想关联的handlers的要一致
        b、只有tasks任务执行成功才会执行handlers中的语句
2. 示例
---
- hosts: testhost
  remote_user: root
  tasks:
    - name: test copy
      copy: src=/tmp/1.txt dest=/tmp/2.txt
      notify: test handlers
  handlers:
    - name: test handlers
      shell: echo "this message is generated by ansible playbook handlers." >> /tmp/2.txt

# cat /tmp/{1,2}.txt (192.168.3.173)
      aaa
      aaa
      this message is generated by ansible playbook handlers.

# cat /tmp/{1,2}.txt
      cat: /tmp/1.txt: No such file or directory
      aaa
      this message is generated by ansible playbook handlers.
